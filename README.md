# Cryptograph Signer Protocol

**Add watch-approved signing to any iOS wallet — no relay server, no
registry, no permission required.**

The Cryptograph Signer Protocol (CSP) lets any iOS wallet request EVM
signatures from the [Cryptograph](https://cryptograph.watch) watch-side
signer over domain-authenticated Universal Links. Keys never leave the
user's Apple Watch; every signature is approved on the watch's own display,
which shows your wallet's verified domain.

📖 **Spec:** [`docs/SIGNER_PROTOCOL.md`](docs/SIGNER_PROTOCOL.md), published
at [cryptograph.watch/signer-protocol](https://cryptograph.watch/signer-protocol)

## Quick start (wallet side)

```ts
import { createPairRequest, handlePairCallback,
         createSignRequest, handleCallback } from '@perpetua/signer-protocol';

// 1. Pair — opens the Cryptograph app; user consents on iPhone + Apple Watch
const { url, pending } = createPairRequest(
  'https://links.yourwallet.example/cryptograph-callback', 'YourWallet');
await persist(pending);            // ALWAYS persist before opening
await Linking.openURL(url);

// 2. Your Universal-Link handler receives the domain-bound callback
const event = handlePairCallback(callbackUrl, pending);
if (event.kind === 'paired') await persist(event.pairing);

// 3. Request a signature — approved on the user's watch
const { url: signUrl, pairing } = createSignRequest(event.pairing, {
  requestId: crypto.randomUUID(), account, chain: 'eip155:1',
  method: 'eth_signTransaction', payload: tx,
});
await persist(pairing);
await Linking.openURL(signUrl);
```

See [`sdk/signer-protocol/README.md`](sdk/signer-protocol/README.md) for the
full API and the two rules that matter (persist-before-open,
verify-every-callback).

## Testing

```bash
cd sdk/signer-protocol
npm install
npm test                    # JCS, HPKE (incl. RFC 9180 vectors), protocol round trips
npm run generate-vectors    # regenerate the deterministic cross-impl fixtures
```

The Swift sources carry their own test suite inside the Cryptograph app
repository, including tests that consume the exact vector file in this repo.
